# Database Design

## Database Type

Relational Database

---

## Database Management System

Microsoft SQL Server

---

## Design Principles

The database was designed using:

- First Normal Form (1NF)
- Second Normal Form (2NF)
- Third Normal Form (3NF)

---

## Primary Keys

Every table contains a primary key to uniquely identify each record.

Examples:

- customer_id
- employee_id
- branch_id
- supplier_id
- product_id

---

## Foreign Keys

Relationships are maintained using foreign keys.

Examples:

- customer_id
- product_id
- supplier_id
- branch_id
- employee_id

---

## Benefits

- Eliminates duplicate data
- Improves consistency
- Improves reporting
- Improves scalability
- Maintains referential integrity
